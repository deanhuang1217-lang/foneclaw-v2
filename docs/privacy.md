# FoneClaw Privacy & Security

## Privacy Model

FoneClaw is built on three privacy principles:

1. **Transparency** — You see exactly what FoneClaw accesses and why
2. **Control** — You grant and revoke permissions individually
3. **Confirmation** — Sensitive actions require your explicit approval

## What FoneClaw Accesses

| Data Type | When Accessed | Stored? | Shared? |
|-----------|--------------|---------|---------|
| Voice input | When you speak a command | Processed, not stored | Not shared |
| Screen content | When reading screen for actions | Not stored | Not shared |
| Contacts | When resolving contact names | Not stored | Not shared |
| SMS content | When reading/sending messages | Not stored | Not shared |
| Notifications | When summarizing notifications | Not stored | Not shared |
| Calendar | When managing events | Not stored | Not shared |
| Location | When navigating or searching nearby | Not stored | Not shared |

## What FoneClaw Does NOT Do

- **No hidden data collection** — FoneClaw does not collect browsing history, app usage, or personal data beyond what's needed for the current command
- **No data selling** — FoneClaw does not sell your data to third parties
- **No background monitoring** — FoneClaw only processes data when you give a command
- **No cross-app tracking** — FoneClaw does not track your activity across apps
- **No advertising** — FoneClaw does not show ads or use your data for advertising

## Confirmation for Sensitive Actions

FoneClaw requires your explicit confirmation before:

- Sending messages (SMS, WhatsApp, Telegram)
- Making phone calls
- Changing system settings (brightness, volume, Wi-Fi, Bluetooth)
- Creating or modifying calendar events
- Deleting data or notes

You can review exactly what FoneClaw will do before confirming. If you don't confirm, the action is not executed.

## Permission Management

All permissions are managed through Android's standard permission system:

1. **Settings → Apps → FoneClaw → Permissions**
2. You can see which permissions are granted
3. You can revoke any permission at any time
4. Revoking a permission disables related features but does not affect other functionality

## Company Independence

FoneClaw is developed by an independent company. It is:

- **Not owned by** any phone manufacturer (Samsung, Xiaomi, Google, Apple, etc.)
- **Not affiliated with** any data broker or advertising network
- **Not a subsidiary** of any larger technology company
- **Independent** — FoneClaw's only product is the FoneClaw phone assistant

## Security Practices

- **No root required** — FoneClaw uses standard Android APIs and Accessibility Service
- **Permission-based** — Each capability requires explicit user permission
- **Confirmation-based** — Sensitive actions require user confirmation
- **Local processing** — Core phone control runs on your device
- **Open permissions** — You can audit exactly what FoneClaw accesses

## Data Processing

- **Voice commands** are processed to understand your intent. Voice data is not stored after processing.
- **Screen content** is read only when needed to execute a command. Screen data is not stored.
- **Contact names** are resolved locally on your device. Contact lists are not uploaded.
- **AI features** may use cloud processing for complex commands. Your data is processed but not stored by the AI service.

## Frequently Asked Questions

**Q: Does FoneClaw record my voice?**
A: FoneClaw processes your voice command to understand your intent. Voice data is not stored after the command is processed.

**Q: Can FoneClaw read my messages?**
A: FoneClaw can read messages only when you explicitly ask it to (e.g., "Read my latest SMS"). It does not read messages in the background.

**Q: Does FoneClaw share my data with Google?**
A: No. FoneClaw is an independent product and does not share your data with Google or any other company.

**Q: Can I use FoneClaw without internet?**
A: Some basic actions (opening apps, system controls) work offline. AI-powered features (natural language understanding, complex commands) require an internet connection.

**Q: Does FoneClaw work with a VPN?**
A: Yes. FoneClaw works with VPN connections. Some AI features may be affected if the VPN blocks certain services.
